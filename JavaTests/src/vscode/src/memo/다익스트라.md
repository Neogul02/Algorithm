## Dijkstra 최단경로 알고리즘

### MST = Minimum Spanning Tree (최소 신장 트리)
-> 그래프에서 **모든 정점**에 대해 최소 비용을 찾는 방법
- 무향 그래프 = 양방향 그래프에서 적용 가능
A->B->C->D 의 최소 비용

### 최단경로 = Shortest Path 
-> 그래프에서 임의의 **두 정점**에 대한 최소 비용을 찾는 방법
- 유향 그래프에서 사용됨
A->D 까지의 최소 비용


가중치가 있는 그래프가 있을때, 최단 경로를 구하는 알고리즘으로 다익스트라 알고리즘

- 음수 가중치는 없어야 함

- 거리 배열을 사용함 초기에는 모든 정점의 거리를 MAX_VALUE로 초기화
- 거리 배열에서 시작 정점의 값은 0으로 초기화 = 거리가 0이다 = 시작정점만 거리가 0이다 

dist[] : 최단 거리 기록을 저장함, 정점 수만큼의 크기로 고정,
priortyQueue : 다음 방문할 정점을 선택, 동적으로 변함, 매 반복마다 최단 거리를 가진 정점을 선택하기 위해 사용

1. PQ에서 (거리, 정점) 꺼냄
2. 꺼낸 정점이 이미 방문한 정점이면 무시
3. 꺼낸 정점이 방문하지 않은 정점이면 방문 처리
4. 꺼낸 거리가 현재 dist[정점]보다 크면, 이미 더 짧은 경로가 발견된 것이므로 무시
5. 꺼낸 정점의 인접한 정점들을 탐색
   - 인접한 정점의 거리를 계산 (꺼낸 거리 + 가중치)
   - 계산된 거리가 dist[인접한 정점]보다 작으면, dist[인접한 정점]을 업데이트하고, PQ에 (계산된 거리, 인접한 정점) 추가


다른 알고리즘으로는 벨만-포드 알고리즘이 있음.
-> 음수 가중치가 있는 그래프에서도 최단 경로를 구할 수 있지만, 시간 복잡도가 O(V*E)로 다익스트라보다 느림
